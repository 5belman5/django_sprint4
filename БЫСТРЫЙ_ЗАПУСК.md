# Быстрый запуск проекта Blogicum

## Правильная последовательность команд в PowerShell:

```powershell
# 1. Перейдите в папку django_sprint4
cd "C:\Users\USER\OneDrive\Рабочий стол\blogicum\django_sprint4"

# 2. Активируйте виртуальное окружение (если нужно)
.\venv\Scripts\Activate.ps1

# 3. Установите зависимости (если еще не установлены)
pip install -r requirements.txt

# 4. Перейдите в папку blogicum (где находится manage.py)
cd blogicum

# 5. Примените миграции
python manage.py migrate

# 6. Создайте суперпользователя (опционально)
python manage.py createsuperuser

# 7. Запустите сервер
python manage.py runserver
```

## Важные замечания:

1. **requirements.txt** находится в папке `django_sprint4`, а не в `blogicum`
2. **manage.py** находится в папке `blogicum` (внутри `django_sprint4`)
3. В PowerShell для активации venv используйте: `.\venv\Scripts\Activate.ps1`
4. Если venv уже активирован, можно пропустить шаг 2

## После запуска:

Откройте браузер и перейдите на: **http://127.0.0.1:8000/**

## Остановка сервера:

Нажмите `Ctrl+C` в терминале

## Запуск тестов:

```powershell
# Перейдите в папку django_sprint4 (где находится pytest.ini)
cd "C:\Users\USER\OneDrive\Рабочий стол\blogicum\django_sprint4"

# Убедитесь, что установлены все зависимости (включая pytz)
pip install -r requirements.txt

# Запустите все тесты
pytest

# Запуск с подробным выводом
pytest -vv

# Запуск конкретного теста
pytest tests/test_post.py

# Запуск конкретной функции
pytest tests/test_post.py::test_post_created_at
```

**Всего в проекте: 25 тестов**

**Важно:** Если видите ошибку `ModuleNotFoundError: No module named 'pytz'`, выполните:
```powershell
pip install pytz
```
